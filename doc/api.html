package routing_table

routing_table

Go 1.21+ · License MIT · Source github.com/mellowdrifter/routing_table

Overview

Package routing_table provides a highly optimised, concurrent, in-memory BGP Routing Information Base (RIB) for both IPv4 and IPv6. It is designed to ingest and query a full Internet routing table (>900k IPv4 prefixes, >200k IPv6 prefixes) with minimal memory overhead and maximum lookup throughput.

⚡
O(1) Root Dispatch
The first 8 bits are resolved with a direct array index, skipping all trie traversal for the outermost octet.
🔀
Add-Path Native
Every trie node stores a map[uint32]*RouteAttributes, keyed by BGP Path ID. Multiple paths per prefix at no extra structural cost.
🗜
Attribute Deduplication
AS Paths, Communities, and Large Communities are globally interned in a ref-counted hash table. Up to 50% memory reduction vs naïve storage.
🔒
Split-Lock Concurrency
Separate sync.RWMutex per address family. IPv4 writes never block IPv6 reads and vice versa.

Installation

$ 
go get github.com/mellowdrifter/routing_table

The only stdlib dependency is net/netip (Go 1.18+). No third-party packages required.

Quick start

goimport (
    "net/netip"
    rt "github.com/mellowdrifter/routing_table"
)

func main() {
    rib := rt.GetNewRib()

    // Build a Route with attributes
    route := rt.Route{
        Prefix: netip.MustParsePrefix("8.8.8.0/24"),
        Attributes: &rt.RouteAttributes{
            AsPath:    []uint32{15169, 3356},
            LocalPref: 100,
            Communities: []uint32{3356<<16 | 22},
        },
        PathID: 0, // 0 = non-Add-Path session
    }

    rib.InsertIPv4(route)

    // Longest-prefix match
    result := rib.SearchIPv4(netip.MustParseAddr("8.8.8.8"))
    if result != nil {
        fmt.Println(result.Prefix)              // 8.8.8.0/24
        fmt.Println(result.Attributes.AsPath)    // [15169 3356]
        fmt.Println(result.Attributes.LocalPref)  // 100
    }
}

Add-Path session example

go// Multiple paths for the same prefix (BGP Add-Path)
rib.InsertIPv4Batch([]rt.Route{
    {Prefix: netip.MustParsePrefix("1.1.1.0/24"), PathID: 101,
     Attributes: &rt.RouteAttributes{AsPath: []uint32{13335, 3356}, LocalPref: 200}},
    {Prefix: netip.MustParsePrefix("1.1.1.0/24"), PathID: 102,
     Attributes: &rt.RouteAttributes{AsPath: []uint32{13335, 174},  LocalPref: 100}},
})

// Retrieve best path (highest LocalPref wins)
best := rib.SearchIPv4(netip.MustParseAddr("1.1.1.1"))

// Retrieve ALL paths for further policy decision
allPaths := rib.AllPathsSearchIPv4(netip.MustParseAddr("1.1.1.1"))
winner := rt.SelectBest(allPaths)

// Delete one specific path
rib.DeleteIPv4(netip.MustParsePrefix("1.1.1.0/24"), 101)

Full table reset (e.g. on peer reset)

go// Atomically drop the entire table and release all memory to GC.
// Faster than deleting prefixes individually.
rib.Reset()

Memory Architecture

Understanding the two-layer storage model is key to reasoning about memory usage in a long-running daemon.

Layer 1 — Root Arrays (O(1) dispatch)
ipv4Root [256]*node
indexed by addr[0]
ipv6Root [32]*node
indexed by addr[0]-0x20
↓
Layer 2 — Binary Trie (bits 9 → max)
node { children[2]*node, paths map[uint32]*Attrs, parent *node }
↓
Layer 3 — Path Map (per prefix node)
PathID 0 → *RouteAttributes
PathID 101 → *RouteAttributes
PathID 102 → *RouteAttributes
↓ (pointer, not copy)
Layer 4 — Dedup Attribute Table (global, ref-counted)
attrTable map[uint64][]*RouteAttributes
RefCount: N (shared by N paths with identical attrs)

Per-struct memory cost

Structure Fixed bytes Variable component Notes
node ~64 B paths map overhead 2×pointer (children) + pointer (parent) + map header
RouteAttributes ~40 B 4 B × len(AsPath) + 4 B × len(Communities) + 12 B × len(LargeCommunities) Stored once per unique attribute set regardless of how many paths reference it
ipv4Root array 2,048 B — 256 × 8-byte pointer; always allocated
ipv6Root array 256 B — 32 × 8-byte pointer; always allocated
attrTable map entry 8 B key + ~24 B bucket per-bucket slice FNV-1a 64-bit key; collision chains are slices of pointers

Real-world estimate (full Internet table, no Add-Path)

Address family Prefix count (approx) Node count (approx) Heap footprint (approx)
IPv4 950,000 1.2 – 1.5 M ~160 – 200 MB
IPv6 220,000 300 – 450 K ~40 – 60 MB
Attribute table ~80–120k unique attribute sets ~20 – 35 MB

With attribute deduplication the attrTable typically holds 1 entry per unique AS-path + community combination, not one per route. For a full Internet table where many thousands of routes share the same AS path (e.g., transit paths via AS 3356), the savings compound significantly — commonly 40–50% less heap vs storing attributes inline.

attrTable internals

Attributes are hashed with FNV-1a 64-bit over all fields (AS path, communities, large communities, LocalPref). On insert, the library deep-copies the caller's struct exactly once and stores a pointer shared by all referencing paths. refCount is decremented on DeleteIPv4/DeleteIPv6; when it reaches zero the entry is evicted and its slice memory reclaimed. The table is protected by its own sync.Mutex, independent of the per-family RIB mutexes.

GC pressure

Reset() re-assigns the root arrays in a single locked step, severing all trie nodes and attribute table entries from GC roots simultaneously. The GC then reclaims the entire abandoned graph asynchronously. This is substantially faster than batch-deleting all routes and avoids the per-deletion attrTable lock contention. Calling runtime.GC() immediately after Reset() is a valid strategy for BGP peer resets when you need memory returned to the OS promptly.

Constraints

These are hard limits enforced at insertion time. Out-of-range prefixes are silently rejected with a log message.

  • IPv4 prefix length: /8 minimum, /24 maximum
  • IPv6 prefix length: /8 minimum, /48 maximum
  • IPv6 address space: 2000::/3 only (first byte 0x20–0x3F; global unicast)
  • PathID 0 is the conventional value for non-Add-Path sessions
  • A nil Attributes pointer is treated as an empty RouteAttributes{} (zero values for all fields)

Types

type Rib struct { … }
Rib is the primary data structure: a concurrent, memory-efficient Routing Information Base holding both IPv4 and IPv6 prefixes in separate binary tries. The IPv4 trie uses a [256]*node root array indexed by the first octet; the IPv6 trie uses a [32]*node root array indexed by firstByte - 0x20, covering all of 2000::/3. Both tries are protected by separate sync.RWMutex instances. Internal fields are not exported; interact via methods only. 
// All exported fields are accessed via methods. Internal layout:
type Rib struct {
    v4mu       *sync.RWMutex
    v6mu       *sync.RWMutex
    ipv4Root   [256]*node              // O(1) first-octet dispatch
    ipv6Root   [32]*node               // O(1) first-byte dispatch (2000::/3)
    attrTable  *attrTable              // global dedup table
    v4Count    int
    v6Count    int
    v4NodeCount uint64
    v6NodeCount uint64
    v4masks    map[int]int
    v6masks    map[int]int
}
type Route struct
Route is a single entry in the RIB. It carries the prefix, all associated BGP attributes, and the Add-Path path identifier. PathID should be 0 for non-Add-Path sessions. 
type Route struct {
    Prefix     netip.Prefix
    Attributes *RouteAttributes
    PathID     uint32  // 0 for non-add-path
}

Route also implements fmt.Stringer — String() returns Prefix.String() or "<nil>".

type RouteAttributes struct
RouteAttributes holds the BGP path attributes associated with a route. The struct you pass to Insert methods is deep-copied once into the deduplicated attribute table; you can discard or reuse your own copy immediately after the call.

The hash and refCount fields are managed internally and must not be set by callers. 
type RouteAttributes struct {
    AsPath           []uint32           // ordered list of ASNs
    Communities      []uint32           // encoded as (ASN << 16 | value)
    LargeCommunities []LargeCommunity   // RFC 8092
    LocalPref        uint32

    // internal — do not set
    hash     uint64
    refCount uint32
}
type LargeCommunity struct
LargeCommunity represents a BGP Large Community as defined in RFC 8092. Each component is an independent 32-bit value.
type LargeCommunity struct {
    GlobalAdmin uint32
    LocalData1  uint32
    LocalData2  uint32
}
type PrefixWithID struct
PrefixWithID pairs a prefix with its Add-Path PathID for batch deletion operations.
type PrefixWithID struct {
    Prefix netip.Prefix
    PathID uint32
}

Functions & Methods

Constructors
func GetNewRib() Rib
GetNewRib creates and returns a new, empty RIB. The root arrays are zero-initialised (all nil pointers); trie nodes are allocated on demand during the first insert into each prefix slot. Also initialises the global attribute deduplication table. 
rib := rt.GetNewRib()
✓ thread-safe after construction
func SelectBest(routes []Route) *Route
SelectBest applies a deterministic BGP best-path selection algorithm to a caller-supplied slice of routes, typically the result of AllPathsSearchIPv4 or AllPathsSearchIPv6. Selection criteria in priority order:
  • Highest LocalPref
  • Shortest AsPath length
  • Lowest PathID as final tiebreaker
Returns nil if the slice is empty.
paths := rib.AllPathsSearchIPv4(netip.MustParseAddr("8.8.8.8"))
best  := rt.SelectBest(paths)
no locks acquired
Rib — Lifecycle
func (r *Rib) Reset()
Reset atomically flushes the entire RIB. Both address-family write locks are acquired simultaneously, the root arrays are replaced with empty arrays, all counters zeroed, and a fresh attribute table installed. The abandoned trie nodes and attribute structs become GC-eligible immediately after the locks are released.

This is the canonical way to handle a BGP peer reset. For memory-sensitive daemons, following Reset() with an explicit runtime.GC(); debug.FreeOSMemory() will return heap pages to the OS promptly.
⚑ acquires v4+v6 write locks
func (r *Rib) PrintRib()
PrintRib prints a short human-readable summary to stdout: IPv4 and IPv6 prefix counts, and the mask distribution maps. Acquires read locks on both families. Intended for debugging; not suitable for programmatic consumption — use V4Count, V6Count, and GetSubnets instead.
read-locked
Rib — Counters & Stats
func (r *Rib) V4Count() int
V4Count returns the total number of distinct IPv4 prefixes currently in the RIB (regardless of how many paths each prefix has). Returns 0 if the RIB has not been initialised via GetNewRib.
read-locked
func (r *Rib) V6Count() int
V6Count returns the total number of distinct IPv6 prefixes in the RIB.
read-locked
func (r *Rib) V4PathCount() int
V4PathCount returns the total number of individual IPv4 paths across all prefixes. For a non-Add-Path session this equals V4Count(). For Add-Path sessions it is ≥ V4Count(). Performs a full trie walk under read lock — do not call in hot paths.
read-locked O(nodes) — avoid in hot path
func (r *Rib) V6PathCount() int
V6PathCount is the IPv6 equivalent of V4PathCount. Performs a full trie walk.
read-locked O(nodes) — avoid in hot path
func (r *Rib) GetSubnets() (map[int]int, map[int]int)
GetSubnets returns copies of the prefix-length distribution histograms for IPv4 and IPv6 respectively. Each map key is a prefix length (e.g. 24), value is the count of prefixes at that length. Returns nil, nil if the RIB was not properly initialised. 
v4masks, v6masks := rib.GetSubnets()
fmt.Printf("/24 count: %d\n", v4masks[24])
read-locked (both families)
Rib — Insert
func (r *Rib) InsertIPv4(route Route)
InsertIPv4 adds a single IPv4 route. If a route with the same prefix and PathID already exists its attributes are replaced (old attributes are ref-count decremented in the attrTable). Silently rejects prefixes outside /8–/24. Ignores routes whose address is IPv6-mapped.
⚑ v4 write lock mask /8–/24 only
func (r *Rib) InsertIPv4Batch(routes []Route) []netip.Prefix
InsertIPv4Batch inserts multiple IPv4 routes under a single write lock acquisition, avoiding per-route lock overhead. Returns a slice of prefixes that were newly added to the trie (i.e., went from 0 paths to 1 path). Useful for tracking which prefixes to announce downstream.

Prefer this over repeated InsertIPv4 calls when processing BGP UPDATE messages with multiple NLRIs. 
newPfxs := rib.InsertIPv4Batch(routes)
for _, p := range newPfxs {
    announce(p)  // only newly visible prefixes
}
⚑ v4 write lock (once) ✦ batch preferred
func (r *Rib) InsertIPv6(route Route)
InsertIPv6 adds a single IPv6 route. Rejects addresses outside 2000::/3 and prefix lengths outside /8–/48. Ignores IPv4 addresses.
⚑ v6 write lock mask /8–/48, 2000::/3 only
func (r *Rib) InsertIPv6Batch(routes []Route) []netip.Prefix
Batch variant of InsertIPv6. Acquires the v6 write lock once. Returns newly-inserted prefixes.
⚑ v6 write lock (once) ✦ batch preferred
Rib — Delete
func (r *Rib) DeleteIPv4(prefix netip.Prefix, pathID uint32)
DeleteIPv4 removes a specific path (identified by pathID) from the given IPv4 prefix. If the prefix has remaining paths it stays in the trie. If no paths remain, the trie node is removed and empty parent nodes are pruned upward. The corresponding RouteAttributes refCount is decremented; the attribute is evicted from the global table when it reaches zero.

Use pathID = 0 for non-Add-Path sessions.
⚑ v4 write lock
func (r *Rib) DeleteIPv4Batch(prefixes []PrefixWithID) []netip.Prefix
Batch variant of DeleteIPv4. Acquires the v4 write lock once. Returns prefixes that were completely removed from the trie (all paths gone), useful for driving downstream withdrawals. 
withdrawn := rib.DeleteIPv4Batch([]rt.PrefixWithID{
    {Prefix: netip.MustParsePrefix("1.1.1.0/24"), PathID: 101},
    {Prefix: netip.MustParsePrefix("2.2.2.0/24"), PathID: 0},
})
for _, p := range withdrawn {
    withdraw(p)  // prefix is fully gone
}
⚑ v4 write lock (once) ✦ batch preferred
func (r *Rib) DeleteIPv6(prefix netip.Prefix, pathID uint32)
IPv6 equivalent of DeleteIPv4.
⚑ v6 write lock
func (r *Rib) DeleteIPv6Batch(prefixes []PrefixWithID) []netip.Prefix
Batch variant of DeleteIPv6. Acquires the v6 write lock once. Returns fully-removed prefixes.
⚑ v6 write lock (once) ✦ batch preferred
Rib — Longest Prefix Match
func (r *Rib) SearchIPv4(ip netip.Addr) *Route
SearchIPv4 performs a Longest Prefix Match for the given IPv4 address and returns the single best-path route. The best path within the matching prefix node is selected by bestPath() using the same criteria as SelectBest.

The returned *Route is freshly allocated; the embedded *RouteAttributes is a direct pointer into the deduplicated table — do not modify it. Returns nil if no covering prefix is found or if ip is IPv6. 
r := rib.SearchIPv4(netip.MustParseAddr("8.8.4.4"))
if r != nil {
    fmt.Println(r.Prefix, r.Attributes.AsPath)
}
read-locked ◈ longest prefix match
func (r *Rib) AllPathsSearchIPv4(ip netip.Addr) []Route
AllPathsSearchIPv4 performs an LPM lookup and returns all paths stored at the matching node, one Route per PathID. Returns nil if no covering prefix exists. Combine with SelectBest to replicate BGP decision process, or inspect all paths directly for policy purposes. 
paths := rib.AllPathsSearchIPv4(netip.MustParseAddr("8.8.4.4"))
fmt.Printf("%d paths available\n", len(paths))
read-locked ◈ longest prefix match
func (r *Rib) SearchIPv6(ip netip.Addr) *Route
SearchIPv6 performs an LPM lookup for the given IPv6 address. Only addresses within 2000::/3 (first byte 0x20–0x3F) are supported; all others return nil. Returns the best-path route or nil.
read-locked ◈ longest prefix match
func (r *Rib) AllPathsSearchIPv6(ip netip.Addr) []Route
IPv6 equivalent of AllPathsSearchIPv4. Returns all paths at the LPM-matching node.
read-locked ◈ longest prefix match
Rib — Exact Prefix Lookup
func (r *Rib) LookupIPv4(prefix netip.Prefix) *Route
LookupIPv4 performs an exact prefix match. Unlike SearchIPv4, it walks the trie to the specific depth indicated by the prefix mask and returns a route only if one exists at that exact node. Returns nil for any less-specific covering prefix.

Use this when you need to check whether a specific prefix is present in the table, e.g. for origin validation. 
// Only returns a route if 8.8.8.0/24 is exactly in the RIB.
// A covering /20 would NOT match here.
r := rib.LookupIPv4(netip.MustParsePrefix("8.8.8.0/24"))
read-locked ◇ exact match
func (r *Rib) LookupIPv6(prefix netip.Prefix) *Route
IPv6 equivalent of LookupIPv4. Walks the trie to the exact prefix depth and returns a route only on exact match.
read-locked ◇ exact match

Concurrency notes

The RIB uses two independent sync.RWMutex instances — v4mu and v6mu. This means concurrent readers for one address family are never blocked by a writer for the other. The attribute table (attrTable) carries its own sync.Mutex, acquired briefly during attribute lookup/insert/release inside write-locked operations.

Safe patterns: simultaneous IPv4 lookups from many goroutines; concurrent IPv4 write + IPv6 read; concurrent IPv4 batch inserts (serialised by v4mu) while IPv6 batch deletes proceed in parallel.

Avoid: calling Reset() from a goroutine while another goroutine holds a reference to a *RouteAttributes pointer returned by a previous search — the pointer remains valid (GC won't collect a live reference), but the data it describes is no longer in the active RIB.

Do not mutate the *RouteAttributes pointer returned by search/lookup methods. It points directly into the shared deduplication table; mutation would corrupt other routes sharing the same attribute set.
